# wx-shuati-xiaochnegxu

一个基于微信小程序的本地刷题工具，支持 JSON 题库导入、顺序练习、错题练习、收藏题目、题目备注、本地题库管理和题库回收站。

本项目的核心目标是提供一个轻量级、可扩展、无需后端服务即可运行的本地刷题小程序模板。用户可以按照指定 JSON 格式制作题库，并通过小程序导入后进行练习。

## 项目定位

本项目不是某个具体考试的题库仓库，而是一个通用的刷题小程序框架。

当前开源版本只提供：

- 小程序源码
- 页面结构
- 工具函数
- JSON 题库模板
- AI 生成 JSON 的提示词规范
- 自编示例题库
- 使用说明

本仓库不提供任何真实考试题库、教材题库、培训机构题库、PDF 原件或从 PDF 中提取出的完整题库数据。

请使用者确保自行导入的题库内容来源合法。

## 项目构建思路

本项目采用“题目本体”和“学习记录”分离的设计方式。

### 1. 题目本体

题目本体保存在题库 JSON 中的 `questions` 字段中，主要包含：

- 题目 ID
- 科目 ID
- 章节 ID
- 题号
- 题型
- 题干
- 选项
- 正确答案
- 解析

题目本体适合长期保存和导入导出。

### 2. 学习记录

学习记录保存在 `records` 字段中，主要包含：

- 答题记录
- 错题记录
- 收藏记录
- 备注记录
- 顺序练习进度
- 错题练习进度
- 收藏夹练习进度
- 删除 / 回收站记录
- 学习统计

这样设计的好处是：

- 题目内容和用户学习数据相互独立
- 题库可以单独导入
- 学习记录可以随题库包一起备份
- 后续接入 WebDAV、云数据库或多设备同步时更容易扩展

### 3. 本地优先

当前版本采用本地优先设计：

- 题库保存在本地文件或本地缓存中
- 学习记录保存在本地缓存中
- 不依赖服务器
- 不需要登录
- 不需要云函数

这种方式适合个人学习、小型题库练习和本地调试。

## 项目构建思路
当前开源版本不需要用户登录，不主动上传用户数据。
题库和学习记录主要保存在本地。使用者应自行管理本地数据和导入题库内容。
请不要在源码中写入：

- API Key
- WebDAV 密码
- 微信 appSecret
- 云环境密钥
- 个人账号信息
- 受版权保护的真实题库
